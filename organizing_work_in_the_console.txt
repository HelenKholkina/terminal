1) alias - создает синонимы других команд. Удобно, когда часто используешь какую-то длинную команду с различными ключами.
Синтаксис: alias синоним = "команда"
Пример: alias // = "ls -la"
        alias cdwork = "cd /home/pingvinusi/Documents/data"


2) source - выполняет все команды из прочитанного файла в текущей командной оболочке.
Пример: soutce /etc/profile


3) Переменные окружения - спец. переменные, определенные оболочкой и используемые программой во время выполнения.
Есть системные переменные (определяются системой для программ системного уровня) и пользовательские.
Общий набор переменныех:
    User - пользователь, использующий систему
    Home - домашний каталог текущего пользователя
    PATH - список каталогов, в которых расположены исполняемые файлы
    PS1 - основная строка приглашения (то, что первое вылетает в терминале при запуске)

Как узнать значение переменной: echo $PATH


4) bashrc - сценарий оболочки, который Bash запускает каждый раз, когда он запускается. Можно поместить любую команду
в этот файл. Сюда же можно посемтить свои переменные для постоянной работы.
