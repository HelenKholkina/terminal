1) vim - текстовый редактор, с которым можно работать через комнадную строку. Во многих
интеграторах уже есть, но иногда может понадобиться установка.
   vim test.txt - открывается test.txt в "обычном режиме" (в нем можно просмотреть файл
или стереть. Переходы в другие режимы из обычного:
   i - переход в "режим ввода"
   ; - переход в "командный режим" (операции с файлом,поиск,замена,настройка редактора)
   esc - выход из любого режима
Команды "обычного режима":
   :q - выход без сохранения
   :wd - сохранить и выйти
Вызов справки о помощи: man vim


2) nano - тоже текстовый редактор. Часто используется для редактирования конфигурацион
ных файлов, когда GUI отсутствует.Запуск:
   * nano - включается редактор, создается файл пустой, при закрытии назвать
   * nano имя_файла - создается файл с заданным именем
   * nano имя_существующего_файла - когда файл уже есть
Управление в nano происходит с помощью "горячих клавиш": Ctrl = "^" 
   ^+G - вывод информации по всем "горячим клавишам"
   ^+C - вернуться к курсору
   ^+W - поиск текста в файле
   ^+X - выход без сохранения
   ^+K - вырезать строку
   ^+U - вставить строку
   ^+6 - включение режима копирования произвольного текста, перемещая курсор, выделяем текст
   Alt+6 - скопировать текст
   ^+S - сохранить и выйти 
   ^+O - сохранить, но не выходить


3) cat - вывод содержимого файлов. Синтаксис: cat опции имя_файла(ов). Опции: -n - вывести
нумерацию строк. Пример:
   * cat file1 file2 - выводит содержимое двух файлов


4) tail - для отслеживания лог-файлов. Синтаксис: tail опции имя_файла(ов). Опции:
   * по умолчанию, без опций, выведет 10 последних записей
   * -f - будет выводить вновь появившиеся логи, но если вдруг произойдет ротация файла, то 
   tail с этой опцией продолжит отслеживать старый лог-файл
   * -F - поэтому лучше пользоваться этой опцией, которая при переименовании лог-файла
   и создании нового лог-файла будет следить за новым
