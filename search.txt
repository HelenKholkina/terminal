1) find - для поиска файлов. Синтаксис: find путь опции критерии_поиска действия_над_файлами. Разберем по отдельности:

А. путь - как относительные, так и абсолютные пути; может быть один путь, может и несколько через пробел. / - корневой
каталог, . - текущая директория.
Б. опции - через тире:
   -depth - поиск в подкаталогах перед поиском в самом каталоге
   -nount - не искать в каталогах других файловых систем
В. критерии_поиска - может быть несколько:
   -atimeN - последний раз к файлу обращались N  дней назад
   -mtimeN - последний раз файл изменяли N дней назад
   -name имя_файла
   -size +/-N[c\w\b\k\M\G] - размер больше (+) или меньше (-), чем N (c-байт, k-килобайт, M-мегабит, G-гигабайт)
   -type C - тип файла (f - файл, d - директория)
   -user имя_пользователя - файл принадлежит
* Критерии можно объединить с помощью операторов: -not - не; -and - и; -or - или. Скобки тоже можно использовать.
Г. действия_над_файлами
   -exec команда \; (команда - rm, например) - выполнит любую команду без вопросов
   -print - вывод имени файла на экран
   -ok - команда\; - тоже выполняет указанную команду, но спрашивает
Привмер:
"find . -size +1000c -type f -print" - вывести на экран (print) обычные файлы (type f) размером больше, чем 1000 байт,
в текущую директорию (.)
"find . -name "~*" -type f -ok rm" - удалить (ok rm) обычные файлы (type f), имя которых начинается на ~(-name "~*")
с предварительным запросом на подтверждение (-ok) в текущей директории (.)
"find / \(-name "~*" -or -name "temp*"\) -type f -print" - вывести на экран (-print) обычные файлы из корневого каталога,
которые начинаются на ~ или на temp


2) grep - поиск по текстовому файлу. Синтаксис: grep опции шаблон имя_файла. Ищет строки с заданным шаблоном. Опции:
   -i - игнорирование регистра
   -v - показать только несовпавшие строки, т.е. строки, которые не содержат шаблон
   -c - выводить количество строк с шаблоном
   -R - рекурсивный поиск, т.е. поиск по всем файлам и каталогам заданной директории. Пример: grep -R test1 one/
